论文阅读笔记——面向航电软件需求的形式化建模与精化研究（未完待续）

发表于2021-06-03|更新于2021-08-02|学习论文阅读
|字数总计:2k|阅读时长:12分钟|阅读量:

这是我师姐的毕业论文，导师希望我好好看看，并在此基础上进行改进，以达到研究成果螺旋式上升的美好愿景。

Reference

[1] 杨琛琛. 面向航电软件需求的形式化建模与精化研究[D].华东师范大学,2021.

Contributions

（1） 这篇论文提出了一个面向航电多层次需求的形式化建模语言S$^2$MSM，该语言抽取SysML 和 MARTE 建模语言的安全子集，并结合安全自动机理论，支持航电高层需求和低层需求的分析和验证。

（2） 构造了S$^2$MSM的形式化语法和语义，使用标签迁移系统作为S$^2$MSM的语义模型，构造了S$^2$MSM中建模元和语义环境的形式化语义。

（3） 构造了从 S$^2$MSM高层需求模型到低S$^2$MSM层需求模型的精化映射关系和精化映射规则。

（4） 设计并实现了基于 S$^2$MSM 的形式化建模工具 S$^2$M，以支持使用S$^2$MSM对安全关键软件需求进行形式化建模与验证。

Related Instructions

这一小节我列出了一些相关定义，方便论文阅读。

航电各层次需求： 高层需求往往关注的是系统的主要功能以及各个部件之间的交互和控制；低层需求则包含了大量与实现相关的细节信息。

MBSE: （基于模型的系统工程）一种形式化的软件开发方法，基于相应的建模语言，MBSE 通过自动或者半自动地将需求模型转换为代码的方式来开发系统，具有良好的可验证性和自动化程度。

MARTE: (Modeling and Analysis of Real-time and Embedded System，嵌入式实时系统的建模与分析)，是 UML 对实时性的扩 展，其时间模型可以用来建模不同类型的时钟和时钟关系。

CCSL： （Clock Constraint Specification Language，时钟约束规范语言)，MARTE 的子语言，能对时钟之间的关系进行定义，提供时钟表达式和时钟关系来指定系统的时间要求和因果依赖关系。

SysML/MARTE： 基于图形描述的建模语言，是国际安全软件工业 MBSE 实践中采用的主流建模语言，被广泛应用于规约安全关键系统的需求。但是，它采用自然语言来定义语义使得模型缺乏系统的、严谨的语义定义。

S$^2$MSM: 本文构造的建模语言，在SysML/MARTE的基础上，结合安全自动机理论，具有精确的语法语义定义。

形式化建模： 在严格的数学定义的基础上，用形式化语言描述软件系统的规范，是对系统进行形式化验证的先行活动。

精化演算： 精化是将抽象程序转换为具体程序的过程，还要保证模型间的一致性。

Theoretical Basis

这一小节是这篇论文的预备知识，介绍了多形态时钟的定义以及 CCSL 中时钟依赖关系的定义，以及SysML/MARTE 中行为状态机 BSM 的相关概念。为了保证航电软件的安全性，论文还参考了 SCADE 中集成的安全状态机 SSM。

部分知识未曾接触过，有些晦涩难懂，慢慢看吧。

多形态时钟：

用于建模多形态时间，不同的系统模块会有不同的时间模型。

多形态时钟的核心是时钟和时钟依赖。在MARTE规范中，时钟被表示为满足全序关系的时钟序列。时钟约束则代表了时钟之间的依赖关系。多个时钟和时钟约束构成了整个系统的时钟系统。

时钟约束规范语言CCSL：

CCSL 提供时钟表达式和时钟关系来指定系统的时间要求和因果依赖关系，所有的系统行为应该都受到 CCSL 约束的限制。

SysML/MARTE行为状态机BSM：

BSM: Behavior State Machine

建模元包括：状态、状态内行为、状态延时触发器、区域、伪状态、结束状态、子状态机、迁移、触发器、卫士条件、迁移行为等。

安全状态机SSM：

SSM：Safe State Machine

SSM理论引自于SCADE，以提高系统建模的安全性， 致力于编程控制为主的软件或硬件系统的建模。

SSM 产生离散输出信号来响应输入信号，状态机则根据特定的瞬时 (instant) 进行响应。瞬时来源于 SCADE 中的同步假设，

在此假设中，时间被划分为离散的瞬时，而不是物理时间。

S$^2$MSM

安全关键软件形式化建模语言S$^2$MSM。

该语言以SysML/MARTE中的行为状态机 BSM 以及 SCADE 中的安全状态机 SSM 为基础，支持多层次需求的建模，特别是支持同步计算模式下的安全状态机建模，保证了系统执行过程中的安全性。

与 BSM 相比， S$^2$MSM 具备以下特性：

(1) S$^2$MSM 关注安全性，是一种确定的状态机，即正确的 S$^2$MSM 的行为迁移和结果有且仅有一种，以杜绝产生模型的二义性。

(2) 支持 SysML 用户使用习惯的连续性。S$^2$MSM 兼容部分 SysML 语法定义及相应的图形化表示，并根据自身需要扩展了部分建模元素。

(3) 确定化 BSM 语义。为了支持建模安全关键的反应式系统，S$^2$MSM 去掉了 BSM 语义中可能导致不确定性的方面。

S$^2$M

本文提出的 S2MSM 语言抽取了 SysML 和 MARTE 建模语言的安全子集，并与基于同步假设的安全状态机理论相结合，支持航电软件需求的建模和验证，以确保系统安全。但是，要将该语言应用到实际案例中，需要设计和开发一个工具，实现 S2MSM 语言的建模和验证功能。

在开源建模工具 Modelio 和开源验证工具 Z3 的基础上，设计并实现了一个工具 S2M 支持 S2MSM 语言的建模和验证。S2M (Safety_SysML Modeling Tool，S2M) 是以 S2MSM 语言规范为基础，支持使用 S2MSM 语言进行系统建模与验证的工具平台，可应用于安全关键系统需求阶段的形式化建模与验证。其原理是利用 S2MSM 语言建模系统需求，并通过模型验证发现需求中的缺陷，例如需求状态模型中的约束冲突、约束违反检查等。

模型分析

该功能是 S2M 的核心功能，模型的形式化过程采用 Java 语言编码实现，独立的后端验证引擎采用 C++ 设计编码实现。主要功能包括：需求模型的完整性分析、需求模型的一致性分析和需求模型的符合性分析。
